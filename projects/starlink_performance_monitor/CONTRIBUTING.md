# Contributing to Starlink Performance Monitor

Thank you for your interest in contributing to the Starlink Performance Monitor project! We welcome contributions from the community to help improve this tool for monitoring Starlink satellite internet performance.

## How to Contribute

### Reporting Issues

If you find a bug or have a feature request, please create an issue on GitHub with:
- A clear and descriptive title
- A detailed description of the problem or enhancement
- Steps to reproduce the issue (if applicable)
- Information about your environment (OS, Python version, etc.)

### Code Contributions

1. Fork the repository
2. Create a new branch for your feature or bug fix
3. Make your changes
4. Add tests if applicable
5. Ensure all tests pass
6. Submit a pull request with a clear description of your changes

### Coding Standards

- Follow PEP 8 style guidelines
- Write clear, descriptive commit messages
- Include docstrings for all functions and classes
- Add type hints where appropriate
- Write unit tests for new functionality

### Testing

Before submitting a pull request, please ensure:
- All existing tests pass
- New functionality is covered by tests
- The code follows the project's style guidelines

Run tests with:
```bash
python -m pytest test_monitor.py -v
```

## Development Setup

1. Clone your fork of the repository
2. Create a virtual environment
3. Install dependencies: `pip install -r requirements.txt`
4. Run tests to ensure everything works: `python test_installation.py`

## Code of Conduct

Please be respectful and constructive in all interactions. We are building a tool to help people better understand their Starlink internet performance, so let's work together positively.

## Questions?

If you have any questions about contributing, feel free to create an issue or contact the maintainers directly.

Thank you for helping improve Starlink Performance Monitor!
